user data

User data is garbage collected when unused. This is not a problem for c, but might be a problem for C++, because deconstructor is automatic invoked, a solution is needed. GC supports to register a finalizer.

size of user_data does not including the room for cls object, e.g.

struct point { int x; int y};
voba_make_user_data(NIL, sizeof(struct point))
((struct point*) user_data_base(v))->x = 100;
((struct point*) user_data_base(v))->y = 100;

see cls.h, which is itself a user defined object.
